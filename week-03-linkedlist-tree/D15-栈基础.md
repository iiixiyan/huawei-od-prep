## 1. 有效的括号 ✅ 高频
**Leetcode 20 · T150**

**问题**：给定一个只包含 `()[]{}` 的字符串，判断括号是否有效匹配。

**思路**：遇到左括号入栈，遇到右括号检查栈顶是否匹配，匹配则出栈，不匹配直接返回 false。最后栈为空才合法。

```python
def isValid(s: str) -> bool:
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in pairs:          # 右括号
            if not stack or stack[-1] != pairs[ch]:
                return False
            stack.pop()
        else:                    # 左括号
            stack.append(ch)
    return not stack
```

**易错点**：栈为空时遇到右括号、遍历结束后栈非空、`pairs` 用右括号做 key 的写法。

**变种**：括号的分数（856）、删除最外层括号（1021）、括号生成（22）。

## 2. 简化路径 ✅
**Leetcode 71 · T150**

**问题**：将 Unix 风格绝对路径简化为规范路径（无 `.`、`..`、连续斜杠）。

**思路**：按 `/` 分割，遍历各部分：空串和 `.` 忽略；`..` 弹出栈顶；其他入栈。最后拼接 `'/' + '/'.join(stack)`。

```python
def simplifyPath(path: str) -> str:
    stack = []
    for part in path.split('/'):
        if part in ('', '.'):
            continue
        if part == '..':
            if stack:
                stack.pop()
        else:
            stack.append(part)
    return '/' + '/'.join(stack)
```

**易错点**：`/../` 结果为 `/` 而非空；多个 `/` 产生空串需忽略。

**变种**：简化路径的 Go 实现（注意路径结尾无 `/`）。

## 3. 最小栈 ✅
**Leetcode 155 · T150**

**问题**：设计一个栈，支持 push, pop, top, getMin，且 getMin 需 O(1)。

**思路**：辅助栈同步保存当前最小值。push 时如果新值 ≤ 当前最小则入辅助栈；pop 时如果弹出值等于辅助栈顶则弹出辅助栈。

```python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, val: int) -> None:
        self.stack.append(val)
        if not self.min_stack or val <= self.min_stack[-1]:
            self.min_stack.append(val)

    def pop(self) -> None:
        val = self.stack.pop()
        if val == self.min_stack[-1]:
            self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]
```

**易错点**：`<=` 而非 `<`（重复最小值时避免丢失）。也可以用 O(1) 空间存差值，但面试推荐辅助栈。

## 4. 逆波兰表达式求值 ✅
**Leetcode 150 · T150/O**

**问题**：根据逆波兰表示法（后缀表达式）求值。

**思路**：遍历 tokens，数字入栈；遇到运算符弹出两个数字运算后结果入栈。注意 `-` 和 `/` 的顺序（`b-a` 而非 `a-b`）。

```python
def evalRPN(tokens: list[str]) -> int:
    stack = []
    ops = {'+', '-', '*', '/'}
    for t in tokens:
        if t in ops:
            b, a = stack.pop(), stack.pop()
            if t == '+': stack.append(a + b)
            elif t == '-': stack.append(a - b)
            elif t == '*': stack.append(a * b)
            else: stack.append(int(a / b))  # 向零取整
        else:
            stack.append(int(t))
    return stack[0]
```

**易错点**：除法向零取整（Python `int(a/b)` 对正负数都向零）；运算顺序（先弹出的是右操作数）。

**变种**：基本计算器 II（227，中缀转后缀）。

## 5. 移除星号 ✅
**Leetcode 2390 · L75**

**问题**：给一个字符串 `s`，`*` 表示删除左侧最近的字符（且星号本身也被删除）。

**思路**：栈模拟即可，遇到 `*` 出栈（栈非空），否则入栈。

```python
def removeStars(s: str) -> str:
    stack = []
    for ch in s:
        if ch == '*':
            if stack:
                stack.pop()
        else:
            stack.append(ch)
    return ''.join(stack)
```

**易错点**：题目保证操作有效（不会在空栈时 `*`），但防御性判断无妨。

**变种**：移除相邻重复（1047）、比较含退格的字符串（844）。

## 6. 小行星碰撞 ✅
**Leetcode 735 · L75/O**

**问题**：行星有大小（绝对值）和方向（正=右，负=左）。**只有** 右行向左行的行星会碰撞（`右、左`），小的消失，相等则同归于尽。

**思路**：栈模拟。遍历行星，当前行星 `x` 向左（负）且栈顶向右（正）时碰撞。栈顶绝对值小则弹出，相等则弹出且当前行星消失，当前行星小则直接消失。否则入栈。

```python
def asteroidCollision(asteroids: list[int]) -> list[int]:
    stack = []
    for x in asteroids:
        while stack and x < 0 < stack[-1]:   # 只有 右→左 才碰撞
            if stack[-1] < -x:
                stack.pop()
                continue
            elif stack[-1] == -x:
                stack.pop()
            break   # 当前行星被摧毁，跳出 while
        else:       # while 未执行或正常结束 → 无碰撞
            stack.append(x)
    return stack
```

**易错点**：碰撞条件 `x < 0 < stack[-1]` 而非绝对值比较；同归于尽后 `continue` / `break` 的选择；星号多次碰撞（栈可能弹出多个）。

**变种**：行星碰撞 II（多个方向叠加的复杂版本）。

## 📦 今日总结

| 题型 | 核心技巧 | 标志 |
|------|---------|------|
| 括号匹配 | 左入右出，查配匹 | 成对出现 |
| 路径简化 | 按分隔符拆分，栈模拟 | `..` 弹出，`.` 忽略 |
| 最小栈 | 辅助栈同步最小值 | O(1) getMin |
| 表达式求值 | 数字入栈，遇符号弹出两个运算 | 后缀表达式 |
| 相邻消除 | 栈顶匹配则出栈 | 星号/退格/相邻重复 |
| 带条件碰撞 | 只有特定方向组合才触发 | 右→左碰撞 |

> 💡 **栈题心法**：遇到"最近关联""成对消除""路径回溯"类问题时，优先考虑栈。写代码时注意边界（空栈、遍历结束后的栈状态）。
