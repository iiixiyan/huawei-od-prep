# Day 39: 贪心 — 贪心的商人 & 活动选择

## 📖 知识点
**贪心算法核心**：
- **局部最优 → 全局最优**：每一步选择当前最优解
- **区间调度**：按结束时间排序，选最早结束的
- **最大利润（股票）**：只要今天价格 > 昨天价格，就累加差价（允许多次交易）
- **分配问题**：排序后双指针或贪心分配
- **证明方法**：交换论证法（exchange argument）或归纳法

## 🧩 刷题任务

### 题目1：贪心的商人 (OD 100分, E卷)
**难度**：⭐⭐
**题目描述**：
你是一个商人，知道未来 N 天某个商品的每日价格。你每天最多只能进行一次操作（买入或卖出），且必须卖完才能再买。求最多能获得多少利润。

**输入**：
```
8
2 5 3 6 1 4 7 3
```
**输出**：
```
12
```
（解释：第1天买2→第2天卖5赚3，第3天买3→第4天卖6赚3，第5天买1→第7天卖7赚6，总利润12）

**思路分析**：
1. 核心：只要能赚钱就交易（今天价格 < 明天价格）
2. 遍历价格数组，如果 prices[i] < prices[i+1]，累加差值
3. 这个解法本质是允许多次交易的最大利润

**参考代码**：
```python
def max_profit(prices):
    profit = 0
    for i in range(len(prices) - 1):
        if prices[i] < prices[i + 1]:
            profit += prices[i + 1] - prices[i]
    return profit

# 测试
prices = [2, 5, 3, 6, 1, 4, 7, 3]
print(max_profit(prices))  # 12
```

**OD备考提示**：这是华为OD高频题！注意允许多次交易时，直接累加所有上升段即可。如果限制交易次数（比如最多2次），则需要 DP 或前后缀分解。

### 题目2：租车骑绿岛 / 分组最少车辆 (OD 100分)
**难度**：⭐⭐
**题目描述**：
公司组织骑绿岛活动，有 N 个人，每辆双人自行车最多载两人，且车有承重限制 limit（两人体重之和 ≤ limit）。问最少需要多少辆自行车？

**输入**：
```
limit = 3
weights = [1, 2, 1, 3, 2, 1]
```
**输出**：
```
3
```
（解释：配队方式 [1+2],[1+2],[1+3] 或 [3],[2+1],[2+1]）

**思路分析**：
1. 排序 + 双指针贪心（经典"救生艇"问题）
2. 将体重从小到大排序
3. 左指针指向最轻的，右指针指向最重的
4. 如果能凑一对（轻+重 ≤ limit），左右指针都移动；否则重的人单独一辆车，只移动右指针
5. 直到左右指针相遇

**参考代码**：
```python
def min_bikes(limit, weights):
    weights.sort()
    left, right = 0, len(weights) - 1
    count = 0
    while left <= right:
        if left == right:
            count += 1
            break
        if weights[left] + weights[right] <= limit:
            left += 1
            right -= 1
        else:
            right -= 1
        count += 1
    return count

# 测试
limit = 3
weights = [1, 2, 1, 3, 2, 1]
print(min_bikes(limit, weights))  # 3
```

**OD备考提示**："救生艇"双指针贪心模板题。注意边界条件：只有一个人时也需要一辆车。排序是 O(NlogN)，双指针是 O(N)。

## 📝 今日小结
- 股票利润（多次交易）：累加所有上坡段差值
- 双指针贪心分配：排序后最小+最大配对
- 区间调度：按结束时间排序，选最早结束的
