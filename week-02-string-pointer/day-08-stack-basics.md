## 📚 知识点：栈（Stack）

### 核心特性
| 操作 | 描述 | 时间复杂度 |
|------|------|-----------|
| `push(x)` | 将元素 x 压入栈顶 | O(1) |
| `pop()` | 弹出栈顶元素 | O(1) |
| `top()/peek()` | 查看栈顶元素 | O(1) |
| `is_empty()` | 判断栈是否为空 | O(1) |

### Python 中栈的两种实现

**1. 列表模拟（推荐）**
```python
stack = []          # 初始化
stack.append(1)     # push
stack.append(2)
top = stack[-1]     # peek
val = stack.pop()   # pop
empty = len(stack) == 0  # 判空
```

**2. collections.deque（双端队列）**
```python
from collections import deque
stack = deque()
stack.append(1)     # push
top = stack[-1]     # peek
val = stack.pop()   # pop
```

> ⚡ **OD 考点**：笔试中 90% 的情况用 Python 列表模拟栈即可，简单高效。

## 🧩 题目 1：从字符串中移除星号

**LeetCode 2390. Removing Stars From a String**

### 问题描述
给你一个包含 `*` 的字符串 s，每次操作可以：
1. 选择一个 `*`
2. 移除它 **以及** 它左边最近的非星号字符

返回移除所有星号后的结果。

**示例**
```
输入: s = "leet**cod*e"
输出: "lecoe"
解释: 
  leet**cod*e → lee*cod*e → lecod*e → lecoe

输入: s = "erase*****"
输出: ""
```

### 思路分析
🚀 **栈模拟 — 经典消除游戏**

遍历字符串：
- 遇到非 `*` 字符 → 入栈
- 遇到 `*` 字符 → 栈顶出栈（消除左边最近字符）

这个过程天然符合栈的 LIFO 特性。

### 图解
```
s = "leet**cod*e"

l → 栈: [l]
e → 栈: [l, e]
e → 栈: [l, e, e]
t → 栈: [l, e, e, t]
* → 弹出 t → 栈: [l, e, e]
* → 弹出 e → 栈: [l, e]
c → 栈: [l, e, c]
o → 栈: [l, e, c, o]
d → 栈: [l, e, c, o, d]
* → 弹出 d → 栈: [l, e, c, o]
e → 栈: [l, e, c, o, e]

结果: "lecoe" ✅
```

### 代码实现
```python
def removeStars(s: str) -> str:
    stack = []
    for ch in s:
        if ch == '*':
            if stack:          # 栈非空则弹出
                stack.pop()
        else:
            stack.append(ch)
    return ''.join(stack)
```

**复杂度分析**
- 时间复杂度：O(n)，n 为字符串长度
- 空间复杂度：O(n)，最坏情况栈中存放所有非星号字符

## 🧩 题目 2：行星碰撞

**LeetCode 735. Asteroid Collision**

### 问题描述
给定一个整数数组 `asteroids`，表示一行小行星。

- **绝对值** 表示行星大小
- **符号** 表示运动方向（正数 → 右，负数 → 左）
- 所有行星速度相同

碰撞规则：
1. 两个行星相向而行才会碰撞（左行行星在右，右行行星在左 → 不会碰撞）
2. 碰撞时，大的摧毁小的
3. 大小相等 → 两者都毁灭
4. 同向而行 → 不会碰撞

返回碰撞结束后的最终状态。

**示例**
```
输入: asteroids = [5, 10, -5]
输出: [5, 10]
解释: 10 和 -5 碰撞 → 10 存活，5 还在

输入: asteroids = [8, -8]
输出: []
解释: 8 和 -8 相等 → 都毁灭

输入: asteroids = [10, 2, -5]
输出: [10]
解释: 2 和 -5 碰撞 → -5 存活，然后 10 和 -5 碰撞 → 10 存活
```

### 思路分析
关键洞察：**只有「右行行星在左，左行行星在右」时才会碰撞**，即栈顶为正数且当前为负数。

用栈模拟：
1. 遍历每个行星
2. 当前行星向右（正数）→ 直接入栈
3. 当前行星向左（负数）→ 与栈顶行星反复碰撞，直到：
   - 当前行星被摧毁 → 跳过
   - 栈顶行星被摧毁 → 弹出，继续检查
   - 栈为空或栈顶也为负数 → 当前行星入栈

### 代码实现
```python
def asteroidCollision(asteroids: list[int]) -> list[int]:
    stack = []
    
    for ast in asteroids:
        # 当前行星向右，直接入栈
        if ast > 0:
            stack.append(ast)
            continue
        
        # 当前行星向左，需要处理碰撞
        # 只要栈顶为正数（右行）且小于当前绝对值，栈顶被摧毁
        while stack and stack[-1] > 0 and stack[-1] < -ast:
            stack.pop()
        
        # 碰撞后检查结果
        if stack and stack[-1] > 0 and stack[-1] == -ast:
            # 相等 → 都毁灭
            stack.pop()
        elif not stack or stack[-1] < 0:
            # 栈为空 或 栈顶也是左行 → 当前行星入栈
            stack.append(ast)
        # 否则：栈顶为正数且 > 当前绝对值 → 当前行星被摧毁，什么都不做
    
    return stack
```

**复杂度分析**
- 时间复杂度：O(n)，每个行星最多入栈一次、出栈一次
- 空间复杂度：O(n)

## 📝 核心套路总结

### 栈的三大应用场景

| 场景 | 典型问题 | 关键信号词 |
|------|---------|-----------|
| **消除/匹配** | 移除星号、括号匹配、删除相邻重复 | "移除"、"消除"、"删除" |
| **碰撞模拟** | 行星碰撞、温度比较 | "碰撞"、"大小比较"、"方向" |
| **历史回溯** | 撤销操作、括号展开 | "最近"、"上一个"、"撤销" |

### 栈解题四步法
1. **确定入栈条件** — 什么情况下元素入栈？
2. **确定出栈条件** — 什么情况下元素出栈？
3. **处理边界** — 栈为空时的处理
4. **构建结果** — 栈中元素如何拼接/转换
